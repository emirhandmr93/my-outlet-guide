# Production Media Policy

This policy defines when outlet media can be considered production-cleared and how new images must be documented before they are added to the app.

## Reuse and sourcing rules

- Do not use official/operator website images unless explicit permission or compatible reuse rights are documented.
- Publicly accessible images are not automatically reusable; availability on the web does not grant production reuse rights.
- New images must have provenance metadata before being added to the repository or referenced by outlet media.
- `sourceStatus: "unknown"` is acceptable only for legacy inventory tracking. It is not acceptable for new production-cleared additions.
- Unknown-provenance local images should be replaced, documented, or excluded before final public production release.
- Prefer project-owned, licensed, public-domain, permission-granted, or controlled CDN/local assets.
- Do not direct-hotlink random remote images or official outlet images.
- Unsplash and Wikimedia assets are acceptable only when the metadata documents the source, author/credit, license or terms, and a non-misleading use note.
- Images should not imply documentary accuracy for a specific outlet unless they are sourced and verified for that outlet.
- Target outlet coverage remains four images (`hero`, `gallery1`, `gallery2`, `gallery3`), with a production-safe minimum of two images per outlet so outlets are not visually empty in production.
- Exact outlet images are preferred; when fewer than two exact production-safe images are available, fill missing slots with production-safe generic non-documentary shopping/outlet visuals.
- Generic visuals must be project-owned, generated by/for the project, licensed, public-domain, or permission-granted, and metadata notes must clearly state that they are generic/non-documentary and do not imply exact outlet accuracy.
- Do not delete old unknown local images for an outlet until safe replacements exist for that outlet; once safe replacements exist for a processed outlet, remove the unknown legacy images for that outlet.

## Project-owned generated generic outlet-atmosphere media

Generated generic imagery is allowed only as a fallback when exact, production-safe outlet images are unavailable or insufficient for an outlet's minimum production coverage. Generated generic visuals are atmosphere fillers, not documentary outlet photos, and they must never claim or imply that the depicted scene is the named outlet.

Generated generic media must follow these rules before it can be promoted as production-cleared:

- The image must be project-created/generated, or otherwise created under terms the project owns or controls.
- The scene must be a generic premium outlet/shopping atmosphere suitable for hero or gallery use.
- The image must not depict the named outlet and must not be described as the named outlet.
- The image must not include real outlet names, operator names, logos, brand marks, readable storefront names, or recognizable brand trade dress.
- The image must not show a recognizable real outlet, mall, landmark, or location.
- People must not be the primary subject; any people should be incidental, non-identifiable, and not person-focused.
- Do not use product-only, object-only, icon, isolated shopping-bag, grocery, dry-cleaning-rack, clothing-pile, or similar non-atmosphere imagery as generated generic outlet media.
- Alt text must describe the generic scene without naming the outlet as the depicted place; prefer wording such as "Generic open-air outlet shopping street with neutral storefronts".
- Metadata notes must explicitly state that the image is generated or project-owned generic media, non-documentary, not an exact image of the outlet, and contains no intended real outlet/operator/brand marks.
- Use `sourceStatus: "project-owned"` when the project owns or controls the generated asset rights.

Recommended metadata conventions for project-owned generated generic images:

- `sourceStatus`: `"project-owned"`
- `sourceUrl`: use an internal, non-remote convention such as `internal:generated-generic/<outletId>/<role>` only after the importer/promoter support internal source identifiers, or omit it only when validation accepts an explicit project-owned/internal note.
- `credit`: `My Outlet Guide / generated project-owned media`
- `license`: `Project-owned`
- `licenseUrl`: omit for project-owned records when tooling allows it; until promotion tooling is updated, any importable manifest will need either a validator-compatible internal source convention or a promotion-tool change because current promotion requires `licenseUrl` in generated metadata.
- `notes`: include the generation batch, human review status, and the required non-documentary disclaimer.

## Required metadata for production-cleared images

Production-cleared image metadata must include:

- `sourceStatus`
- `sourceUrl`, or an explicit note that the image is project-owned/internal
- `credit` when applicable
- `license`
- `licenseUrl` when applicable
- `alt`
- `notes` when needed to explain permission, ownership, context, or non-documentary use

## Image quality recommendations

- Use optimized WebP assets when practical.
- Keep dimensions controlled for the app surfaces where images appear.
- Keep file sizes reasonable for mobile users.
- Provide useful alt text for every image.
